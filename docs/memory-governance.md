# Memory Governance

Memory governance means deciding what systems, teams, and tools are allowed to remember, expose, update, and forget.

## Practical Checks

- Identify what information is being stored.
- Decide who can read, change, export, or delete it.
- Set retention expectations before collection.
- Separate personal memory, organizational records, and public artifacts.
- Review whether AI tools are storing prompts, files, or generated summaries.

## Watch For

- Systems that remember more than users expect.
- Old context being treated as current truth.
- Sensitive details copied into summaries or training examples.
- No way to correct or remove inaccurate memory.

## Human Gate

For sensitive or long-lived records, a responsible person should approve what is retained and how correction requests are handled.
