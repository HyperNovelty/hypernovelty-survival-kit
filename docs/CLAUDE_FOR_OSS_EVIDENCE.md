# Claude for OSS Evidence Notes

Status: truthful evidence note for future application drafting. This file is not an application submission and does not claim eligibility.

## Source criteria summary

The Claude for Open Source page lists several eligibility patterns, including dependency/download scale, recognized core maintainer status, 100+ merged PRs to repos not owned by the applicant, 20+ external contributors in one repo, OpenSSF criticality, and a flexible clause for maintainers of projects the ecosystem quietly depends on.

## Current strengths

- Public repository footprint exists and is thematically coherent.
- Most repos already include README, license, security policy, contributing guide, and code of conduct.
- The current sprint adds CI validation and contribution issue templates across the portfolio.
- Projects use public-safe examples and avoid private data, account actions, and secret-dependent workflows.

## Current gaps

- No dependency/download threshold evidence yet.
- No stars/forks/issues/community signal yet.
- No external contributor threshold evidence yet.
- No claim should be made that these repos are critical infrastructure.
- Some repos still need package metadata, richer demos, releases, and external review.

## Honest future application framing

> I maintain HyperNovelty OSS, a small public portfolio of local-first tools for AI-era maintainer work: agent run receipts, workflow safety screens, source-card schemas, verification literacy kits, education readiness kits, and anti-slop development gates for AI coding agents. These repos are intentionally conservative: synthetic fixtures, deterministic validators, no secrets, no account actions, and human-review gates. They are early and do not yet meet dependency/download thresholds, but they address a growing maintainer pain: making AI-agent contributions and AI-assisted workflows inspectable rather than merely plausible.

## Do not say

- Do not claim the portfolio meets hard Claude for OSS thresholds unless live metrics prove it.
- Do not claim outside dependency, institutional adoption, or critical infrastructure status without evidence.
- Do not claim the tools certify safety, compliance, or correctness.
- Do not imply an application has been submitted from this repository.
